      # 什么是同源策略及限制

      • 协议、域名、端口三者一致，此为同源
      • 不同源的资源文件无法交互，此为限制
      • 跨域下Cookie、LocaleStorage、IndexDB无法获取
      • DOM无法获得
      • Ajax请求无法发送

      # 安全类

      # CSRF

      • 基本概念和缩写
        跨站请求伪造，英文名：Cross-site request forgery 缩写：CSRF
      • 攻击原理
        1. 用户访问网站A
        2. 网站A为用户下发Cookie
        3. 用户访问网站B
        4. 网站B发给用户引诱链接（实际地址为A网站的一个API，比如：支付商品）
        5. 用户点击链接，执行链接的行为，攻击完成
      • 防御措施
        • Token验证：访问接口必须要验证token
        • Referer验证：验证来源页面
        • 隐藏令牌：将一个另外隐藏起来如放到http请求头中

      # XSS

      • 基本概念和缩写
        跨域脚本攻击，英文名：Cross-site scripting
      • 攻击原理
        利用合法渠道，向页面注入恶意script
      • 防御措施
        • 提交时过滤script脚本标签字符

      # 排序

      • 冒泡排序
      • 快速排序
        • 实现思路
          1. 取数组的中间项，及其索引
          2. 遍历数组，小于中间项的放到left数组中，大于中间项的放到right数组中
          3. 拼接left 中间项 right 并把结果递归
          4. 递归结束条件：数组长度为1
        • 源码
          function fastSort(arr) {
    if(arr.length <= 1) {
        return arr
    }
    let index = Math.floor(arr.length / 2)
    let center = arr.splice(index, 1)[0]
    let left = []
    let right = []
    for(let i = 0; i < arr.length; i++) {
        if(arr[i] < center) {
            left.push(arr[i])
        } else {
            right.push(arr[i])
        }
    }
    return fastSort(left).concat([center], fastSort(right))
}

      # 页面渲染机制

      # DOCTYPE有什么用

      • DTD (Document type define) 文档类型定义，是一系列的语法定义，用来定义XML/XHTML的文件类型，浏览器根据其判断文档类型，来决定如何解析。
      • DOCTYPE是用来声明文档类型和DTD规范的，一个主要的用途便是文件的合法性验证。如何文件不合法，那么浏览器解析时会出错。

      # 浏览器的渲染过程

      1. 取到 html 和 css 代码
      2. 使用 Html Parser 和 Css Parser 把取到的代码处理成 Dom Tree 和 Style Rules
      3. 把 Dom Tree 和 Style Rules 整合成 Render Tree
      4. 通过 Layout 计算页面元素应该显示在哪些位置
      5. 通过 Paint 绘制DOM并展示

      # 重排Reflow是什么

      Dom元素都有自己的盒模型，这些都需浏览器根据各个元素按照自己的样式来计算，并按照计算结果把该元素放置在对应的位置，这个过程叫做Reflow
      触发Reflow的一些常见操作：

      • 添加、删除、修改Dom元素
      • 修改CSS
      • 移动DOM元素（动画）
      • Resize窗口的时候
      • 修改网页默认字体的时候
      • ...

      # 重绘Repaint是什么

      Dom元素的样式位置等都确定下来以后，浏览器把元素按照各自的样式绘制了一遍，于是页面内容出现了，这个过程称为Repaint
      触发Repaint的一些常见操作：

      • DOM改动
      • CSS改动

      # 任务队列

      • 同步任务：Javascript 主线程上排队的任务，只有一个执行完了后才能执行下一个
      • 异步任务：当异步任务可以执行了，才会进入主线程执行
        • setTimeout, setInterval, setImmediate等
      • 宏任务：可以理解为每一次执行栈执行的代码（包括异步任务的回调事件）
      • 微任务：当前主任务执行结束后立即执行的任务
        • promise (then, catch, finally), process.nextTick等
      • 执行机制：同步任务 → 微任务 → 异步任务
      console.log(1)
setTimeout(()=> {
    console.log(2)
})

new Promise((resolve)=> {
    console.log(3)
    setTimeout(()=> {
        console.log(4)
    })
    resolve()
}).then(function() {
    console.log(5)
}).finally(function() {
    console.log(6)
})
console.log(7)
// 输出结果：（同步） 1 3 7 （微任务） 5 6 （异步） 2 4

      # 页面性能

      # 提升页面性能的方法有哪些？

      • 资源文件压缩合并，减少Http请求次数
        • 打包合并优化、GZip压缩、精灵图
        • Http请求并发6个
      • 非核心代码异步加载、异步加载的方式、异步加载的区别
        • 异步加载的方式
          1. 动态脚本加载(动态创建script节点)
          2. defer
          3. async
        • 异步加载的区别
          1. defer是在浏览器HTML解析完成 DOMContentloaded 之后才会执行，如果是多个，则按照加载的顺序依次执行，一般用于依赖DOM元素的脚本
          2. async是加载完了以后立即执行，如果是多个，执行顺序和加载顺序无关
      • 利用浏览器缓存、缓存的分类、缓存的原理
        • 缓存的分类
          1. 强缓存(缓存有效时间内不会再次请求服务器，会取浏览器缓存)
            • Expires
              // 服务端设置过期时间，由于服务端与客户端时间可能不一致
Expires:Tue Jan 07 2020 16:43:08 GMT+0800
              1
              2
            • Cache-Control
              // 缓存3600秒（1小时）
Cache-Control:max-age=3600
              1
              2
          2. 协商缓存
            • Last-Modified, If-Modified-Since：
              1. 浏览器第一次和服务器交互时，服务器返回资源会在response header中携带Last-Modified标识资源在服务器的最后一次修改时间
              2. 浏览器再次向服务器请求该资源时，会在request header中携带If-Modified-Since，值为服务器下发的Last-Modified的值
              3. 服务器再次收到资源请求时，根据浏览器传过来的If-Modified-Since和服务器上的最后一次的修改时间做比较，判断是否有变化，如果没有变化返回304 Not Modified，但是不会返回资源内容，如果有变化就返回资源内容，当服务器返回304 Not Modified时，response header不会再添加Last-Modified
              4. 浏览器接收到304响应时，从缓存中加载资源
              5. 如果协商缓存没有命中，浏览器直接从服务器加载资源，Last-Modified会在重新加载的时候被更新，下次请求时，If-Modified-Since也会被更新
            • Etag, If-None-Match
              1. 浏览器请求服务器内容时，服务器返回该内容的同时，在response header中加上Etag标识资源文件是否有修改
              2. 浏览器再次请求这个资源时，在request header加上If-None-Match，值为上次请求返回的Etag的值
              3. 当服务器再次收到请求时，会取到request header中的Etag，并与最新的Etag作比较，相同则返回304 Modified，不会返回资源文件内容，如果不同则返回资源内容。
              4. 浏览器接受到304 Modified响应后，从缓存中加载资源
          3. 一般服务器上的Last-Modified, If-Modified-Since和Etag, If-None-Match会同时开启，协商缓存需要配合强缓存使用。

      • mvvm之双向绑定

        • 正向 data → view
        • 反向 view → data
        • 双向 view ↔ data

    enumerable: true,   // 当该属性值为true时，该属性才能出现在对象的枚举中，才能被遍历到
    configurable: true, // 当该属性值为false时，该属性描述符才能改变，同时该属性也能从对象删除
