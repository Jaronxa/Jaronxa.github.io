        网站性能优化之组件懒加载

        Jaron

        最近做网站性能优化，在当前优化的基础上参考某电商网站，实现了网站首页组件懒加载功能，并且网页性能通过了测试要求。

        # 需求背景

        最近近我们公司网站做性能调优，要求在Google PageSpeed Insights上的网站评分能够达到80分以上。为此，在处理过项目其它方向的优化之后，我决定补充实现组件懒加载的方式来提升性能。目前主流电商网站的首页都会有懒加载组件功能，如：Tmall , JD 等，我们可以在 Chrome 浏览器的 Network 中把网络速度设置为 Slow 3G，然后刷新网页，可以很明显的看到其首屏渲染只有 SEO 部分，然后向下滚动，可以看到页面上的组件在滚动加载。

        # 需求分析

        实现网站的滚动加载组件功能，需要保证两个前提

        1. 不影响或尽可能小的影响到网站的 SEO
        2. 懒加载交互要求稳定

        为保证此两点要求，我们首先要先确定哪些数据采用 SSR（Serve Side Render 服务端渲染），哪些数据可以在客户端异步获取， 经过分析某电商网站的首页渲染方式，结合我们网站实际技术选型（Laravel + Vue），采用了服务器推页头与骨架结构，客户端异步请求 Blade 组件并渲染的方式。

        # 实现步骤

        整体的实现步骤主要分为以下四个部分

        # 2. 数据缓存

        考虑到服务器压力问题，数据不应该在每一次异步加载组件的时候都去请求全部数据，在这里我们采用了数据缓存，即：当用访问网站时，将获取到的数据缓存到 PHP 服务器。动态加载组件时，直接从缓存中获取数据并渲染。具体的实现如下：

        // 使用页面的完整地址作为缓存Key
$cacheKey = \URL::current();
// 缓存数据
Cache::forever($cacheKey, json_encode((Object)[
    'language' => $this->language,
    'cmsDataObjects' => $cmsDataObjects,
    'cmsCompontents' => $cmsCompontents,
]));

        # 3. 模板获取

        按照之前的设置，网站首屏获取的页面就只保留了头部页面结构，剩下的组件需要我们异步加载，加载的条件为：出现在可视区域内部。我们先来看一下 JS 部分的代码。

        import PubSub from 'pubsub-js'
export default class LazyDom {
    constructor(selector) {
        this.selector = selector
        this.nodeList = Array.from(document.querySelectorAll(selector))
        this.loadingList = []
        this._loadContainerContent()
        this._scrollListener()
    }
    // 滚动监听
    _scrollListener() {
        window.addEventListener('scroll', this._throttle(()=> {
            this._loadContainerContent()
        }));
    }
    // 获取dom
    _loadContainerContent() {
        for(let i = 0; i < this.nodeList.length ; i++) {
            if(this.loadingList.indexOf(i) < 0) {
                let element = this.nodeList[i];
                if(this._getElementTopLeft(element).top + element.clientHeight > window.pageYOffset && window.pageYOffset + window.innerHeight > this._getElementTopLeft(element).top) {
                    this.loadingList.push(i)
                    const key = this.nodeList[i].getAttribute("data-key")
                    const configId  = this.nodeList[i].getAttribute("data-id")
                    const url = `${laroute.route("api.website.index.queryBlade")}?key=${key}&id=${configId}&link=${encodeURIComponent(window.location.href)}`
                    this._ajax("GET", url).then((res)=> {
                        this._draw(res,i)
                        PubSub.publish('domLazyLoad', key)
                    })
                }
            } else {
                continue;
            }
        }
    }
    // 封装Ajax
    _ajax(type, url, data = null) {
        return new Promise(resolve => {
            let xhr
            if(window.XMLHttpRequest) {
                xhr = new XMLHttpRequest()
            } else {
                xhr = new ActiveXObject()
            }
            xhr.open(type, url)
            xhr.send(data ? JSON.stringify(data) : null)
            xhr.onreadystatechange = ()=> {
                if(xhr.readyState === 4) {
                    if(xhr.status === 200) {
                        resolve(xhr.responseText)
                    }
                }
            }
        })
    }
    // 判断元素是否出现在可视区域
    _getElementTopLeft(obj) {
        let top = 0
        let left = 0
        while(obj){
            top += obj.offsetTop
            left += obj.offsetLeft
            obj = obj.offsetParent
        }
        return {top:top,left:left}
    }
    // 节流函数
    _throttle(fn, delay = 300) {
        let timer = null
        return function() {
            if(timer) {
                return
            } else {
                timer = setTimeout(()=> {
                    fn.apply(this, arguments)
                    timer = null
                }, delay)
            }
        }
    }
    // 渲染Dom
    _draw(res, i) {
        let div = document.createElement('div')
        div.innerHTML = res
        this.nodeList[i].parentNode.replaceChild(div, this.nodeList[i])
    }
}

        控制器内部处理请求，并且返回视图模板

        public function queryBlade(Request $request) {
    $data = [];
    $key = $request->get('key');
    $configId = $request->get('id');
    $cacheKey = $request->get('link');
    $cacheData = Cache::get($cacheKey);
    if($cacheData) {
        $cacheData = json_decode($cacheData);
    } else {
        abort(404);
    }

    $language = $cacheData->language;
    $cmsDataObjects = $cacheData->cmsDataObjects;
    $cmsCompontents = $cacheData->cmsCompontents;

    for($i = 0; $i < count($cmsCompontents); $i++) {
        $component = $cmsCompontents[$i];
        if($component->component_config_id === $configId)
        {
            $data = [
                'properties' => $component->properties,
                'datakeys' => $component->dataobjects,
                'datas' => getCmsDatas($component, $cmsDataObjects, $userTagList),  // 获取更多数据
                'configId' => $component->component_config_id,
                'componentsId' => $component->component_key.strval($i)
            ];
        }
    }

    $view = 'cms._cms_component_' . $key;
    return view($view, $data);
}

        # 总结

        经过此次懒加载组件处理，我们网站的首次内容绘制时间、首次有效绘制时间、首次 CPU 闲置时间均已达到一秒以内，并且在 Google PageSpeed Insights 的评分也达到了87分，极大的提升了网站加载速度，并且做到了较好的交互体验。此实践为服务端渲染实现的方式，如果采用的是客户端渲染，可以采用同样的思想，数据缓存，异步渲染，此外，如果有条件的话，最好可以在首次加载的时候不要一次性把整个页面的数据全部取到，最好可以跟服务端配合，做成多接口分次请求获取数据的形式来优化。
